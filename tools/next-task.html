How It Works

1

Write TASKS.md

Define tasks in a simple markdown file. Status, description, context.

2

Run next-task

It finds the next todo, spawns Claude Code, watches it work.

3

AI executes

Claude reads the codebase, makes changes, runs tests. Fully autonomous.

4

Loop or stop

Mark complete, move to next. Or stop when a task fails for human review.

Quick Start

# Create TASKS.md in your project
cat > TASKS.md << 'EOF'
# Tasks

## T1: Add user authentication [done]
- Implement JWT-based auth
- Add login/logout endpoints

## T2: Add rate limiting [todo]
- Limit API to 100 req/min per user
- Return 429 when exceeded

## T3: Add API documentation [todo]
- Document all endpoints in docs/API.md
- Include examples
EOF

# Run next task
next-task run

# Or run until all done
next-task run --loop

# Check status
next-task status
